## v1.17 — Adaptive Context Compaction (ACC)

**Released:** March 2026 | **Versions:** 1.17.0, 1.17.1

The headline feature of v1.17 is **Adaptive Context Compaction** — a 5-stage
system that automatically extends your effective context window as conversation
grows. Previously, ~3,700 lines of context management code were wired up but
never activated. This release brings all of it to life.

### What ACC Does

As your conversation fills up, Gofer now applies graduated compression
automatically instead of just warning you:

| Stage | At  | What happens                                                                         |
| ----- | --- | ------------------------------------------------------------------------------------ |
| 1     | 70% | Logs a delegation advisory — recommends offloading work to sub-agents                |
| 2     | 80% | Older tool outputs (file reads, command results) get replaced with compact summaries |
| 3     | 85% | Budget enforcement kicks in — oversized context sections are trimmed                 |
| 4     | 90% | All observations forced to their most compact form                                   |
| 5     | 99% | Full LLM-based summarization of completed work                                       |

Each stage has a 30-second cooldown and higher stages always supersede lower
ones. All stages are error-safe and will never crash the extension.

### How Observation Masking Works

Gofer tracks tool outputs as "observations" with a 3-tier decay system:

- **Full** — Recent observations kept in complete detail
- **Key-points** — Older observations show only key findings
- **Masked** — Old observations become a ~15 token placeholder (expandable on
  demand via `expand_observation`)

This means a session that would have hit the context wall at 70% can now often
continue to 90%+ before needing a manual save/resume cycle.

### Other Changes in v1.17

- **Memory leak fix** (v1.17.1): ACCOrchestrator is now properly disposed during
  workspace changes, preventing resource exhaustion when switching projects
- **Stronger test assertions** (v1.17.1): Replaced placeholder assertions with
  spy-based verification across 4 test files
- **Dead-code activation**: ContextBuilder, ObservationMasker,
  SubAgentDispatcher, MemoryLayerManager, and StageContextProfileLoader are all
  now wired and executing
- **42 new tests** covering ACC events, orchestrator stages, integration
  scenarios, and wiring verification

## v1.16 — Documentation Site, Business Pipeline, and AI Instructions

**Released:** March 2026 | **Versions:** 1.16.0 through 1.16.6

v1.16 was a broad release that added the documentation website you're reading
right now, expanded the pipeline with business-focused stages, and introduced
automatic AI instruction generation.

### Documentation Site (v1.16.3 — v1.16.5)

Gofer now has a proper documentation site built with Docsify and hosted on
GitHub Pages. This replaced scattered markdown files with a searchable,
navigable website covering the full pipeline, configuration reference, and
guides.

- Pipeline documentation for all 8 stages including the new 0a and 7a stages
- Configuration reference with 30+ documented settings
- Session management guide with ACC documentation
- Auxiliary commands reference
- Integrated releases page that the extension's auto-updater reads from

### Business Consultant Pipeline (v1.16.3)

Two new pipeline stages bracket the technical work:

- **`/0a_problem_validation`** — Before any code is written, validates the
  business problem using 5 Whys root-cause analysis, stakeholder impact mapping,
  and market landscape research. Produces a problem brief that frames the
  technical work.
- **`/7a_stakeholder_comms`** — After validation passes, generates a stakeholder
  communications package: release notes, demo script, change management brief,
  and training outline.

All 37 pipeline agents are now bundled with the extension and automatically
deployed to user projects on init.

### AI Instruction Generation (v1.16.0)

When you run `gofer.initialize` on a repository, Gofer now auto-generates three
AI instruction files tailored to your project:

- **CLAUDE.md** — Instructions for Claude Code
- **AGENTS.md** — Agent configuration
- **copilot-instructions.md** — Instructions for GitHub Copilot

These are generated by analyzing your project structure (language, framework,
test runner) and producing concise, project-specific guidance. The generated
files are kept under 60 lines each (tightened in v1.16.6).

### Other Changes in v1.16

- **Mandatory pipeline steps** (v1.16.4): All pipeline stages are now mandatory
  — no more optional labels that could be skipped
- **Copilot Chat prompts** (v1.16.5): Added `.github/prompts/` files for the new
  0a and 7a stages
- **Compact instruction templates** (v1.16.6): Rightsized CLAUDE.md, AGENTS.md,
  and copilot-instructions.md to under 60 lines each
- **Release verification** (v1.16.2): The release script now verifies
  auto-update availability after publishing

## v1.15 — Pipeline Auto-Chain

**Released:** March 2026 | **Version:** 1.15.0

The big change in v1.15 is that the entire pipeline now runs autonomously with a
single command. Previously, each stage required manual invocation. Now:

```text
/0_business_scenario Add search feature with filters
```

This single command chains through all stages automatically: problem validation,
research, specify, plan, tasks, implement, validate, and stakeholder comms. Each
stage invokes the next via Skill invocation without stopping for approval.

The pipeline only stops if a stage fails validation, giving you a fully
autonomous feature implementation experience.

## v1.14 — Multi-Agent Strategies and Engineering Remediation

**Released:** February — March 2026 | **Versions:** 1.14.0 through 1.14.5

### Multi-Perspective Sub-Agents (v1.14.0)

Each pipeline stage can now spawn multiple sub-agents that analyze the same
problem from different angles using a diverge-converge pattern:

- **Research stage**: Multiple agents explore codebase patterns, dependencies,
  and similar implementations in parallel
- **Validation stage**: Six specialist agents (correctness, security,
  performance, test quality, integration, standards) score independently
- A synthesis step merges diverse perspectives into a unified output

### Engineering Remediation (v1.13.0 — v1.13.6)

A significant internal quality push that doesn't change features but makes
everything more reliable:

- **Dependency injection framework** (tsyringe) for cleaner service management
- **Service extraction**: DisposalService, StateManager, EventHandlers,
  InstructionGenerator pulled out of the monolithic extension.ts
- **Logger bridge**: Unified logging across all components
- **Cache bounds**: Memory-bounded caches to prevent unbounded growth
- **Memory leak fixes**: Duplicate workspace listener registration and missing
  resource cleanup resolved
- **Activation crash fix**: `reflect-metadata` import order, command
  registration timing, and config schema issues all addressed

### Other Changes in v1.14

- **Auto-handoff** (v1.14.3): Save/clear/resume cycle when context hits critical
  threshold
- **Memory panel wiring** (v1.14.3): Memory commands now register after
  MemoryManager creation
- **API key validation** (v1.14.5): Removed overly strict `minLength:1` schema
  that blocked upgrades

## v1.12 — Engineer Review Gate and Dead Code Removal

**Released:** February 2026 | **Versions:** 1.12.0 through 1.12.2

### Engineer Review Gate (v1.12.0)

The `/4_gofer_tasks` stage now includes an automatic engineer review gate that
cross-references the spec, plan, and task breakdown before implementation
begins. This catches misalignments early — before any code is written.

### Dead Code Removal (v1.12.2)

Removed unused code paths accumulated during rapid development:
`spawnNewTerminalFn`, `sendSaveToTerminal`, `sendResumeToTerminal`,
`autonomousMonitoringInterval`, stability tracking, and `maybeNotify`. Cleaner
codebase, fewer maintenance risks.

## v1.11 — Fully Autonomous Execution

**Released:** February 2026 | **Version:** 1.11.0

Removed all inter-stage approval gates. After planning completes and tasks are
approved, the pipeline executes fully autonomously through implementation and
validation. This was a prerequisite for the auto-chain feature in v1.15.

## v1.10 — Auto-Save at Context Threshold

**Released:** February 2026 | **Version:** 1.10.0

Automatic save-and-resume when context reaches 69% utilization. The extension
saves a session checkpoint, clears context, and resumes from the checkpoint —
all without manual intervention. This was later refined by the ACC system in
v1.17.
